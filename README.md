# egon-content

[![Tests](https://github.com/EgonWorkbooks/egon-content/actions/workflows/tests.yml/badge.svg)](https://github.com/EgonWorkbooks/egon-content/actions/workflows/tests.yml)
[![Coverage](https://codecov.io/gh/EgonWorkbooks/egon-content/branch/main/graph/badge.svg)](https://codecov.io/gh/EgonWorkbooks/egon-content)

A command-line tool for mental health professionals to generate ~300-word educational articles as Markdown files, formatted for [Logseq](https://logseq.com) or [Obsidian](https://obsidian.md).
Content is generated by Claude and reviewed before publication.

## Getting started

### Prerequisites

- [uv](https://docs.astral.sh/uv/) — Python package and environment manager
- An [Anthropic API key](https://console.anthropic.com)

Install `uv` if you don't have it:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### Installation

Clone the repository and install dependencies:

```bash
git clone <repo-url>
cd egon-content
uv sync
```

### Configuration

Copy the example environment file and add your API key:

```bash
cp .env.example .env
```

Open `.env` and set your key:

```dotenv
ANTHROPIC_API_KEY=your-api-key-here
```

### Usage

**Generate a single article:**

```bash
uv run main.py generate --app obsidian --topic "managing social anxiety"
uv run main.py generate --app logseq --topic "apprehension"
```

**Generate a full topic pack:**

```bash
uv run main.py pack --app obsidian --pack basic-emotions
uv run main.py pack --app logseq --pack anxiety-toolkit
```

**Generate articles from a topics file:**

```bash
uv run main.py from-file --app obsidian --file my-topics.txt
uv run main.py from-file --app logseq --file my-topics.txt
```

Copy `topics.example.txt` to get started:

```bash
cp topics.example.txt my-topics.txt
```

The file format is one topic per line. Blank lines and lines starting with `#` are ignored:

```text
# Anxiety
Managing social anxiety
Breathing exercises for anxiety

# Low mood
Self-compassion
Building a daily routine
```

**See all available packs and topics:**

```bash
uv run main.py list-packs
```

Output files are saved to `generated_content/obsidian/` or `generated_content/logseq/`, named `<topic-slug>.md`. Existing files are overwritten.

## Reviewing generated content

### Logseq

1. Generate a file with `--app logseq`
2. Copy the file from `generated_content/logseq/` into the `pages/` folder of your Logseq graph
3. Logseq detects the file immediately — the page appears in your graph without any import step
4. Open the page in Logseq to review and edit before publishing

### Obsidian

1. Generate a file with `--app obsidian`
2. Copy the file from `generated_content/obsidian/` into your Obsidian vault folder
3. Obsidian detects the file immediately — the note appears in your vault without any import step
4. Open the note in Obsidian to review and edit before publishing

## Roadmap

The current focus is Logseq and Obsidian. Once the concept is proven, the following may be added:

### Additional app targets

- **Zettlr** — academic/research focus, Zettelkasten methodology, Pandoc export
- **Dendron** — Markdown-based, lives inside VS Code as an extension
- **SiYuan** — block-based like Logseq, self-hostable
- **Anytype** — local-first, peer-to-peer architecture

### Other potential improvements

- Custom pack definitions via config file or CLI (without editing `packs.py`)
- Quality review step (e.g. word count check, safe messaging lint)
