# egon-content

[![Tests](https://github.com/Egon-Notebooks/egon-content/actions/workflows/tests.yml/badge.svg)](https://github.com/Egon-Notebooks/egon-content/actions/workflows/tests.yml)

A command-line tool for mental health professionals to generate and curate educational articles as Markdown files, formatted for [Logseq](https://logseq.com) or [Obsidian](https://obsidian.md).
Content is generated by Claude and should be reviewed before publication.
This repository generates general content, and will never contain any personal information.

## Getting started

### Prerequisites

- [uv](https://docs.astral.sh/uv/) — Python package and environment manager
- An [Anthropic API key](https://console.anthropic.com) — for article text generation
- An [OpenAI API key](https://platform.openai.com/api-keys) — for image generation (optional, use `--no-image` to skip)

Install `uv` if you don't have it:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### Installation

Clone the repository and install dependencies:

```bash
git clone <repo-url>
cd egon-content
uv sync
```

### Configuration

Copy the example environment file and add your API key:

```bash
cp .env.example .env
```

Open `.env` and set your key:

```dotenv
ANTHROPIC_API_KEY=your-api-key-here
```

### Usage

**Generate a single article (with image by default):**

```bash
uv run main.py generate --app logseq --topic "apprehension"
uv run main.py generate --app obsidian --topic "managing social anxiety"
```

**Generate without an image:**

```bash
uv run main.py generate --app obsidian --topic "managing social anxiety" --no-image
```

**Generate the full base collection (all packs):**

```bash
uv run main.py generate-all --app logseq --no-image
uv run main.py generate-all --app obsidian
```

**Generate a single pack:**

```bash
uv run main.py pack --app logseq --pack mindfulness-and-presence
uv run main.py pack --app obsidian --pack anxiety-and-worry
```

**See all available packs and topics:**

```bash
uv run main.py list-packs
```

Output files are saved to `generated_content/obsidian/` or `generated_content/logseq/`, named `<topic-slug>.md`.
If existing files are found, you will be prompted whether you want to overwrite the content.

## Reviewing generated content

### Logseq

1. Generate a file with `--app logseq`
2. Copy the `.md` file from `generated_content/logseq/` into the `pages/` folder of your Logseq graph
3. Copy the `.png` file from `generated_content/logseq/assets/` into the `assets/` folder of your graph
4. Logseq detects both files immediately — no import step needed
5. Open the page in Logseq to review and edit before publishing

### Obsidian

1. Generate a file with `--app obsidian`
2. Copy the `.md` file from `generated_content/obsidian/` into your Obsidian vault
3. Copy the `images/` folder from `generated_content/obsidian/images/` into the same vault folder
4. Obsidian detects the files immediately — no import step needed
5. Open the note in Obsidian to review and edit before publishing
